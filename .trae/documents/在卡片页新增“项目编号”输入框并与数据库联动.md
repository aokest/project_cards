## 目标
- 在 cardv8.html 的左侧“控制台”增加“项目编号”输入框，支持直接编辑项目编号
- 输入后即时更新右侧卡片抬头显示及原始文本 rawText 中的“项目编号：”行
- 同步到数据库时将项目编号作为 cardId，保持时间表联动与过滤逻辑不变

## UI 改动
- 在“控制台”面板的“选择卡片”下方新增一行：
  - Label：项目编号
  - Input：文本框，实时显示并可编辑当前项目编号
  - 占位符：示例 1205-S1-S03

## 数据绑定与更新
- 新增状态 inputCardId，初始化为 parsedData['项目编号']
- onChange：
  - 更新 inputCardId
  - 以正则替换 rawText 中的“项目编号：”行；若不存在则在文本首行插入“项目编号：<新值>”并保留原内容
- parsedData 的解析逻辑保持不变（从 rawText 中提取键值）

## 校验与提示
- 校验格式：^\d{4}-S\d+-[SP]\d{2}$（如 1205-S1-S03 或 1205-S2-P01）
- 若不匹配：
  - 输入框下方提示“编号格式不正确（示例：1205-S1-S03）”
  - 允许保存但在同步数据库前进行二次确认/提示

## 保存与同步
- 本地保存（左下角“保存”）保持原逻辑
- 同步数据库（“同步数据库”按钮）：
  - 使用 parsedData['项目编号'] 作为 cardObj.cardId
  - id 维持原（activeCardId），仅更新 cardId；若无 activeCardId 则生成新 id
  - 同步成功后将 URL 参数 ?id 更新为当前 cardId 以便刷新后仍指向该项目

## 选择卡片交互
- 当从“选择卡片”下拉选择已有卡片时：
  - inputCardId 自动填充为所选卡片的编号
  - 保持样式与模块配置随卡片切换而更新

## 时间表联动
- 时间表页面仍依据 cardId（项目编号）解析阶段与前缀，不需改动

## 兼容性注意
- 代码避免使用反引号模板字符串，采用普通字符串与数组 join
- 不引入新外部库，复用现有 React/Tailwind/Babel 结构

## 测试用例
- 输入有效编号（1205-S1-S03）：右侧抬头立即更新；保存与同步后，时间表显示并可筛选
- 输入无效编号：出现格式提示；同步前弹窗确认
- 切换到其他卡片：编号输入框跟随更新
- 刷新页面：URL 参数保留当前编号并正确加载对应卡片
