# Git 版本控制与工程化维护计划

为了满足您将项目纳入 Git 管理、分离数据与代码、以及支持未来扩展的需求，我制定了以下执行计划：

## 1. 工程化整理
我们将把当前散乱的文件整理成标准的工程结构。
- **依赖管理**：虽然目前使用 Python 内置库，但为了将来扩展（接入 CRM/收入管理），我会创建一个标准的 `requirements.txt`（暂时留空或注明环境要求）。
- **说明文档**：创建 `README.md`，详细说明项目启动方式、目录结构及注意事项。

## 2. 安全与配置管理 (核心需求)
- **凭证分离**：确保代码中无任何硬编码 Key（已检查，当前代码是安全的）。
- **一键导入/导出配置**：
    - 我将在 **AI 设置弹窗** 中增加 **“导出配置”** 和 **“导入配置”** 按钮。
    - 您可以将包含 Key 的配置导出为 `config.json` 文件保存在本地（如 U 盘或安全目录）。
    - 换电脑或重新部署时，只需点击“导入配置”加载该 JSON 文件，即可一键恢复所有 API Key 和设置，无需手动复制粘贴。
- **Git 忽略规则**：创建 `.gitignore` 文件，严格忽略以下文件，确保它们绝对不会被上传到 GitHub：
    - `project_cards.json` (数据库文件)
    - `*.key`, `config.json` (任何可能的密钥文件)
    - `__pycache__` 等临时文件

## 3. Git 仓库初始化与推送
- 初始化本地 Git 仓库。
- 提交所有代码文件（`html`, `py`, `css`, `docs`）。
- 关联远程仓库 `https://github.com/aokest/project_cards`。
- 推送 `main` 分支。

## 执行步骤
1.  **修改前端代码**：在 `cardv8.html` 的 AI 配置面板中实现配置文件的导入/导出功能。
2.  **创建工程文件**：生成 `README.md` 和 `.gitignore`。
3.  **Git 操作**：初始化、提交、推送。

准备好后，请确认执行。
