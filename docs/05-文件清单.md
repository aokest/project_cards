# 文件清单与版本说明

本文档记录项目中关键文件的用途、说明及变更历史，便于后续开发者（人类或AI）快速理解系统结构。

## 核心应用文件

### `index.html`
- **用途**: 系统入口页，项目列表管理仪表盘。
- **功能**: 
  - 展示所有项目卡片摘要。
  - **[v2024-12-24 新增]** 多选/全选卡片。
  - **[v2024-12-24 新增]** 批量导出 Markdown。
  - **[v2024-12-24 新增]** 预算统计与汇总（Modal弹窗）。
  - 全局 AI 配置入口。
  - 数据的导入（JSON）、导出（JSON）与备份。
- **依赖**: React 18, Tailwind CSS, Babel Standalone.

### `cardv8.html`
- **用途**: 项目卡片核心编辑器与生成器。
- **功能**:
  - 16:9 响应式卡片渲染。
  - **[v2024-12-24 优化]** AI 润色流程：增加 `PolishedPreviewModal`（确认弹窗）和 History Undo（撤销）机制。
  - 布局调整（列宽拖拽、字体大小滑块）。
  - 主题配色切换。
  - 数据自动保存至 LocalStorage。
- **依赖**: React 18, Tailwind CSS, Lucide Icons.

## 数据与脚本

### `project_cards.json`
- **用途**: 默认项目数据源。
- **说明**: 包含系统初始化时加载的演示或迁移数据。结构为 JSON 数组，包含 `cardId`, `title`, `rawText` (原始文本), `data` (解析后的键值对) 等字段。

### `convert_projects.py`
- **用途**: 数据迁移工具。
- **说明**: 用于将特定格式（`####` 分隔）的 Markdown 项目文件转换为系统可用的 `project_cards.json`。
- **用法**: `python convert_projects.py`

### `initial_data.json`
- **用途**: 兜底数据。
- **说明**: 当 LocalStorage 和 `project_cards.json` 都为空时加载的示例数据。

## 部署与配置

### `docker-compose.yml`
- **用途**: 容器编排文件。
- **说明**: 定义了 Nginx 服务，映射端口 `18888:80`，挂载当前目录到 `/usr/share/nginx/html`。

### `Dockerfile`
- **用途**: 镜像构建文件。
- **说明**: 基于 `nginx:alpine`，用于构建生产环境镜像。
