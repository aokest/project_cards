# 项目说明文档

## 1. 项目背景
本项目“项目卡片及项目推进表生成器”旨在为项目经理和团队提供一个高效、可视化的工具，用于生成标准化的项目卡片（Project Cards）。这些卡片通常用于展示项目的核心信息（如目标、周期、预算、进度等），便于在汇报、沟通和管理中使用。

## 2. 核心目标
- **标准化输出**：通过统一的模板和数据结构，生成风格一致的项目卡片（16:9 比例）。
- **AI 赋能**：利用 LLM（大语言模型）辅助润色项目描述，提取关键信息，提升内容专业度。提供润色预览与撤销机制，确保内容可控。
- **灵活配置**：支持自定义配色、布局（列宽拖拽）、字体大小，满足不同展示需求。
- **数据管理**：支持项目数据的导入导出（JSON/Excel/Markdown），具备多项目批量管理能力。
- **经营辅助**：提供项目预算的自动提取与汇总统计功能，辅助经营决策。

## 3. 愿景与融合路径
本项目不仅仅是一个静态生成器，未来定位为企业级经营管理系统的核心组件：
- **阶段一（当前）**：单机版工具，支持 Markdown/JSON 导入，卡片可视化编辑与导出，具备基础统计功能。
- **阶段二（近期）**：多项目管理，支持更复杂的预算统计、项目集视图，本地数据持久化优化。
- **阶段三（未来）**：与 CRM（客户关系管理）、商机管理系统融合。
    - 实现从“商机”到“立项”再到“交付”的全生命周期管理。
    - 项目卡片将作为 CRM 中“商机”或“项目”的可视化投影。
    - 数据互通，实时同步预算、进度和风险信息。

## 4. 目录结构说明
- `cardv8.html`: 核心编辑器，单页 React 应用，负责卡片的渲染、编辑、AI 润色（含确认流程）和导出。
- `index.html`: 项目入口与列表页，负责展示所有项目卡片，支持多选、批量导出 Markdown、预算统计等管理功能。
- `docs/`: 项目文档中心，包含需求、设计、进度、文件清单等。
- 关键规范：参见 `docs/01-开发规范.md`（开发流程、数据安全与脱敏策略）。
- `projects/`: 存放原始项目数据（Markdown/JSON）。
- `project_cards.json`: 本地数据源（不提交远端，已忽略）。
- `env/test/project_cards.json`: 测试环境/脱敏数据源（空）。
- `docker-compose.test.yml`: 测试环境 Compose，端口 18898。
- `convert_projects.py`: 数据转换脚本（Markdown -> JSON）。

## 5. 关键特性
- **Smart Balance**: 自动平衡卡片布局权重。
- **AI 辅助写作**: 支持多模型（OpenAI/DeepSeek等），具备润色预览、接受/拒绝及撤销功能。
- **批量管理**: 支持项目多选，一键导出 Markdown 汇总报告。
- **智能统计**: 自动解析非结构化文本中的预算数字，生成汇总清单。
- **交互勾选**: 支持 Markdown 待办事项的交互。
- **Docker 部署**: 支持容器化部署，便于分发。
